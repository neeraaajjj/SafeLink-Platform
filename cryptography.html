Cryptography Roadmap

A guide to the core principles and applications of modern cryptography.

Stage 1: Foundational Concepts & History

Before diving into complex algorithms, you must understand *why* cryptography exists and the basic terminology that defines it.

Key Concepts

  • The CIA Triad: Understand Confidentiality (secrecy), Integrity (data is unchanged), and Authentication (proving identity).
  • Terminology: Learn Plaintext (unencrypted), Ciphertext (encrypted), Key (the secret), Algorithm/Cipher (the method), Encryption, and Decryption.
  • Classical Ciphers: Study the Caesar Cipher and Vigenere Cipher to understand *why* they are weak (e.g., frequency analysis).
  • Basic Math: Get comfortable with Modular Arithmetic ("clock math") and the concept of Prime Numbers.

Stage 2: Symmetric Cryptography (Private Key)

The "workhorse" of cryptography. This type uses a **single shared key** to both encrypt and decrypt data. It's extremely fast and used for bulk data encryption.

Key Concepts

  • Core Principle: One key to lock, the *same* key to unlock.
  • Block Ciphers: Encrypt data in fixed-size blocks. The modern standard is AES (Advanced Encryption Standard).
  • Stream Ciphers: Encrypt data one bit or byte at a time. Examples include ChaCha20.
  • Modes of Operation: Understand *how* block ciphers are used (e.g., GCM - Galois/Counter Mode), which provides both confidentiality and authentication.
  • The Key Problem: How do you securely share the secret key with the other person in the first place?

Stage 3: Asymmetric Cryptography (Public Key)

This revolutionary concept solves the key-sharing problem by using a **pair of keys**: one public (to encrypt) and one private (to decrypt).

Key Concepts

  • Core Principle: A Public Key (share with everyone) and a Private Key (keep it secret!). Data locked with the public key can *only* be unlocked with the private key.
  • Key Exchange: The Diffie-Hellman (DH) algorithm allows two parties to create a shared symmetric key over an insecure channel.
  • Encryption Algorithms: RSA (based on the difficulty of factoring large prime numbers) is the most famous.
  • Elliptic Curve Cryptography (ECC): A newer, more efficient type that provides the same strength as RSA with much smaller keys.

Stage 4: Hashing & Data Integrity

This is not encryption. A hash function is a one-way street that creates a unique, fixed-size "fingerprint" (a hash) of any data. It's used to verify integrity.

Key Concepts

  • Properties: One-way (cannot be reversed), Deterministic (same input always gives same output), and Collision-Resistant (hard to find two inputs with the same hash).
  • Algorithms: Learn why MD5 and SHA-1 are broken. The modern standards are the SHA-2 family (e.g., SHA-256) and SHA-3.
  • Use Cases: Securely storing passwords (hash the password + a "salt"), verifying file downloads.
  • HMAC (Hash-based MAC): A hash function combined with a secret key. This provides *authentication* and integrity (proves *who* sent the data and that it wasn't changed).

Stage 5: Real-World Applications (Putting it all together)

This is how these individual pieces are combined to secure the internet and our communications.

Key Concepts

  • TLS/SSL (HTTPS): The "lock" icon in your browser. It uses Asymmetric crypto (RSA/ECC) for the initial "handshake" to securely exchange a Symmetric key (AES). Then, it uses fast Symmetric crypto for all the website data.
  • Digital Signatures: Provides authentication and non-repudiation. You create a hash of a message and then encrypt the *hash* with your *private* key. Anyone can verify it using your *public* key.
  • PKI (Public Key Infrastructure): The trust system of the internet. How do you know a public key *really* belongs to Google? Certificates (which bind an identity to a public key) and Certificate Authorities (CAs) (the trusted organizations that issue them).

Stage 6: Advanced & Future Topics

The field is constantly evolving to combat new threats and enable new technologies.

Key Concepts

  • Post-Quantum Cryptography (PQC): Developing new algorithms that are secure against attacks from future, large-scale quantum computers.
  • Homomorphic Encryption: A cutting-edge technique that allows performing computations on encrypted data *without* decrypting it first.
  • Zero-Knowledge Proofs: A method to prove that you know a piece of information (like a password) without revealing the information itself.
